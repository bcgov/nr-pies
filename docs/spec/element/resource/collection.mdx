Represents a collection. A collection serves as a common construct for describing data elements that group other data
elements together.

## Scope

This resource type is only transmitted within a Record message by setting `asset_kind` to `PROJECT`. A Collection is
defined as an independent, composable resource type and may be used in other message contexts.

Refer to [Record](/docs/spec/element/message/record) for details on the message envelope pattern and how resource types are selected.

## Structure

This structure contains no properties at this time and is functionally equivalent to an empty object.

The Collection resource type is an empty object and does not define any properties. Systems parsing collections should
ignore any unexpected properties.

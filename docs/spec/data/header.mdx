Standard message header attributes present for all PIES message data types. All defined attributes are needed for
identification and attribution.

## Scope

This data type is intended to define all expected attributes that a message datatype must contain at minimum. This
data structure is not intended for standalone use.

## Structure

| Name | Cardinality | Type | Description & Constraints |
| --- | --- | --- | --- |
| transaction_id | 1..1 | `string` | A unique UUIDv7 assigned for this specific message. |
| version | 1..1 | `string` | The PIES specification version this message complies to. Value shall be a valid semantic version formatted string. |
| kind | 1..1 | `string` | The specific kind of PIES message data type this message represents. |
| system_id | 1..* | `string \| string[]` | A valid URI or set of URIs that identifies the source system, service or asset that the data comes from. This is typically a string; however an array of unordered URIs can be used to indicate that multiple systems were used to compile the data in this message. This URI should point to an authoritative asset tracking system or equivalent. However, the URI does not need to be machine readable. |
| record_id | 1..1 | `string` | The record or primary key representing what the data this message is directly associated to. |
| record_kind | 1..1 | `string` | The kind of record the source system stores this record as (i.e. Permit, Project, Submission or Tracking). |

## Implementation Notes

All header attributes are mandatory. A valid PIES message datagram must define all of the header attributes.
